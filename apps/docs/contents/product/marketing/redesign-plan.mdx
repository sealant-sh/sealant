This is the plan for re-spinning the marketing site onto Sealant's locked product decisions. The existing page is well-built but sells the wrong story: a hosted cloud, umbrella-named products, and a two-sided SaaS funnel. The fix is a content, information-architecture, and copy re-spine on the existing visual system. The design language stays; the narrative changes.

<Callout type="info">
The redesign was produced by exploring five divergent approaches, judging them through three lenses (a first-time developer, a skeptical open-source hacker, and a design steward), then synthesizing one blueprint. The winning spine is **"the missing runtime" manifesto**, with a **one-liner → live-to-recorded hero** grafted in to fix five-second clarity, and **the execution record as the climax**.
</Callout>

## What this redesign optimizes

For an adoption play, the page has exactly two jobs. Everything below serves them. The call to action being optimized is **a GitHub star**, not a signup.

  <Card title="Trust for the skeptic">

## The spine

A builder-to-builder argument: the agent era shipped frameworks that decide *what* to do, but left a hole where the work should happen and a void where the truth should be. **Sealant is the open-source, self-hosted runtime that fills both.**

The page leads with the plain definition, *then* picks the fight, and proves it instantly with a hero panel that types the canonical one-liner and resolves it into a real recorded run. The execution record is the soul and the climax; the one-liner is the literal spine, repeated across many jobs to show versatility.

## Build sequencing and risks

The honest framing: this is **not** a pure re-spine. The emotional payload — the hero live-to-record panel, the signature light inspector, the replay re-fold, and the versatility re-type — is net-new stateful and animated code on a page that today has zero client interactivity. Treated carelessly, it reads as the exact vaporware this audience punishes.

<Steps>
  <Step>
    **Build the signature run-record panel first**, as a reusable component. It is the spine of four sections; everything else is cheaper once it exists.
  </Step>
  <Step>
    **Static-legible first.** Every animation must degrade to a fully-legible static panel under reduced-motion and no-JS. The static panels must carry the argument alone. Consider shipping static and layering motion later.
  </Step>
  <Step>
    **Loud "building now" framing** on the inspector, replay, and Handoff surfaces. If a developer clones the repo and finds replay or Handoff not real, it inverts the page's own honesty thesis. Mocked records are labeled illustrative; the Quickstart promises only what runs today.
  </Step>
</Steps>

Other risks: the manifesto voice can tip into preachy, so every opinion must cash out immediately in copy-pasteable SDK code; thirteen sections risk fatigue, so the hero must carry the wow and the leaner core ships first if needed; cobalt-lift must stay reserved (final CTA at `shadow-md`); and Quickstart correctness is load-bearing and must be verified against the codebase before ship.

## Design direction — "Evidence on file"

This direction came out of running the redesign through a distinctive-design lens (three subject-grounded visual identities, a templated-default critique, then synthesis). It governs the visual identity and overrides the per-section visual notes above where they conflict.

### The realization

Run the templated-default test honestly: for any dev-infra brief, the default hero is a dark terminal code block — which is exactly what the current page is. That dark code panel is the single most templated thing on the page. So distinctiveness does not come from a new palette or font (the warm-canvas, cobalt, three-voice system is a pinned brief — keep it). It comes from finally shipping the one artifact the design language literally names "signature" but the page has never built: the **warm-light run-record exhibit**. No other dev tool can render an "exhibit" because no other dev tool keeps a sequenced, provenance-tagged, replayable record. The signature is the product's own data model, rendered.

### The signature instrument — the run-record exhibit

Build it once as a reusable `@sealant/ui` exhibit component, parameterized by fidelity (full, inline-strip, or inert). It is the hero, the versatility footers, the record climax, the browser variant, and the final closer — and it replaces the dark code panel as the page's main instrument. Four traits, each mined directly from the real data model, that no dark code block has:

<Cards>
  <Card title="Corner seal">
    A cobalt recording pulse, a mono `run-id`, and a capture timestamp. Marks it as filed evidence — the durable telemetry spool plus correlation id.
  </Card>
  <Card title="Sequence rail">
    Mono per-event numbers (`0001 0002 0003 …`) down the left margin, like a ledger — the real monotonic `(runId, sequence)` coordinate.
  </Card>
  <Card title="Provenance as dot-plus-word">
    Observed, Inferred, or Unknown on each fact — the real `captureMethod` confidence field, rendered in the status grammar, never a boxed pill.
  </Card>
  <Card title="Evidence rows and diff peek">
    Hairline mono evidence rows plus a diff peek (2px green/red edge-marks, never floods) and a `Replay ▸` control that re-folds the append-only rows in place — replay as a pure fold of the log.
  </Card>
</Cards>

<Callout type="idea">
Build order is unchanged: still "build the signature panel first" — but the panel is the light exhibit, not a dark terminal. It is the vertical slice everything else depends on.
</Callout>

### Where the dark code panel still lives

Only three places, because in each a terminal is honest: the **SSH** section (a real interactive session), the **Quickstart** (literal copy-paste commands), and the **indictment foil** (the dimmed wall of `agent.log` that fails to parse, shown as the thing the record beats). Everywhere else the dark panel currently appears as a hero or feature visual, it becomes the light exhibit.

### Catalog IA — eyebrows that reference real runs

Structure encodes truth: each section's eyebrow becomes a real artifact reference, not a decorative label — for example `EXHIBIT C · run sbx_8m2k · 184 events`. Hard rule: an eyebrow must reference a real run primitive (run-id plus event count), never a generic word. The page reads as a small, numbered dossier of evidence you could file. The literal `EXHIBIT A–H` lettering is an optional flourish; the run-id reference is always on.

### Typography move

Keep the three voices and the semantic split. The move: mono carries the evidence (sequence numbers, event names, offsets, run-id seal, provenance) inside the exhibit; Inter carries five-second comprehension in the claim that sits beside every exhibit; Space Grotesk names the exhibit, one headline per section, with restraint. The core law — the claim never carries a value the exhibit does not show — is promoted from principle to layout law: reject any section where the claim asserts a value the visible record does not prove. That is "reports, not judges" made structural.

### Palette

100% inherited, zero new hexes. The warm canvas is the gallery wall (exhibits float as objects via soft shadow, never borders); pure white is reserved for the kept-record card; cobalt is the only brand color (the seal and pulse, the active provenance link, the one primary CTA), never flooded. Green-dot Observed dominates, amber marks exactly one Inferred or not-run moment (the browser "DOM assertion: not run" edge, and the boundaries honesty), and red appears only for demonstrated breakage.

### Motion budget — exactly two, static-first as a hard floor

Every record renders fully legible with no JS and under reduced-motion (offsets show final values, the pulse becomes a static dot, replay shows the already-folded rows). Exactly two motions ship:

<Cards>
  <Card title="Hero resolve">
    The hero one-liner resolving into the exhibit.
  </Card>
  <Card title="Sticky run-header playhead">
    A persistent run-header in the sticky nav (run-id, recording pulse, and an advancing `mm:ss.mmm` offset) so scroll subliminally reads as replaying one run — opening on `sandbox.ready` at the hero and closing on `run.completed` at the final CTA, where the pulse stops and status resolves to `Completed · observed`.
  </Card>
</Cards>

No per-section reveals beyond these, no scroll-jacking, no full-height gutter. Ship static; layer the two motions as a fast-follow.

### The one risk, and the one accessory removed

- **The risk (spend boldness here, nowhere else):** the sticky run-header playhead. Justified because it is the product's actual data model used as navigation, not metaphor — but it must stay hairline-quiet, single-accent, fully static-degradable, and never required to understand what Sealant is. Removable without breaking the page if it does not land.
- **Removed (the disciplined cut):** the deckled, torn-paper top edge on the record card. It is kitsch and fights the "depth is one restrained soft shadow, no decorative chrome" rule. The "kept artifact" read is already carried by the seal, rail, and provenance dots. The record should look filed, not distressed.

### Net deltas to the plan

- The main instrument is the run-record exhibit, not the dark code panel.
- Eyebrows become real run references (`run <id> · <N> events`).
- The record build is the exhibit component with the four required traits above.
- Add the sticky run-header playhead (ambient, optional, static-degradable).
- The motion budget is exactly two motions; static-first is a hard floor, not a recommendation.
