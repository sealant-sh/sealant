1 · Architecture (Docker; k3s is unused)

Sandboxes run as Docker containers via packages/sandboxes/src/runtime/docker-runtime-adapter.ts. sealantd runs inside the container, listening on a Unix socket. The orchestrator reaches it from outside the container; tonight via docker exec, later via the SSH gateway (a tunnel channel).

3 · Decisions & tradeoffs

  • Docker only. k3s/k8s adapters are stubs. sealantd runs in the container; no new RuntimeAdapter id needed (but L5 service is adapter-agnostic so the gateway/k8s path can slot in later).
  • GHCR image, not bundled binary. ghcr.io/get-sealant/sealantd:0.1.2 is public + multi-arch (amd64/arm64) + verified pullable. COPY --from = production path, no build-context plumbing. Pin 0.1.2 → 0.1.3 after that release clears. Offline fallback binary at .sealantd-work/sealantd.
  • docker exec transport for now. The SSH gateway is an SSH relay; reaching an in-pod unix socket needs a new tunnel channel (deferred). docker exec proves the SDK path realistically and is what the worker can already do.
  • Relay must be added. No base image (fedora/arch/nixos) ships socat/nc-with-unix. Add socat to the sealantd-enabled image's packages (fallback: bundle a static socat for nixos/distroless).
  • peer-cred. sealantd validates SO_PEERCRED (same uid / root / allowlist). docker exec runs as a uid → must match sealantd's uid, or set --allowed-peer-uids, or run both as the sandbox user. Will verify + handle.

B · Key decisions & tradeoffs

  • Default-off flag → zero blast radius: existing sandboxes render identically; sealantd is fully opt-in per build.
  • docker exec + socat transport for now (real + what the worker can already do). The SSH-gateway tunnel is the production transport — deferred.
  • Local SDK link via root pnpm.overrides (file:../sealantd/packages/*) because fromStream isn't published yet. Swap to a released SDK once it ships.
  • peer-cred: root (uid 0) is always allowed; container + docker exec both run as root → no --user needed. Enforcement proven (unauthorized uid is rejected even with the socket world-writable).
  • Env: disabled Docker Desktop's containerd snapshotter (UseContainerdSnapshotter=false) to unblock image pulls — left in place (reverting re-breaks pulls).
